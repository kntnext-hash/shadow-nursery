# Codex Guide

このワークスペースでCodexに頼むときの使い方メモです。

## まず何を頼めばいいか

基本は日本語でそのまま頼んで大丈夫です。

例:

```text
AGENTS.mdとPHASE_CURRENT.mdを読んで、次の作業を進めて
Phase 1を実装して。終わったらビルド確認までして
今の進捗をPHASE_CURRENT.mdに反映して
このSwiftファイルをレビューして
ビルドエラーを直して
次にやるべきことを短く整理して
```

## このプロジェクトで最初に読ませるファイル

Codexに作業させる前に、次のように頼むのがおすすめです。

```text
AGENTS.md、PHASE_CURRENT.md、SCHEMA.md、CONTENT.mdを読んでから作業して
```

各ファイルの役割:

- `AGENTS.md`: プロジェクト全体の仕様、フェーズ、禁止事項
- `PHASE_CURRENT.md`: 現在のフェーズ、完了条件、進捗メモ
- `SCHEMA.md`: データモデル、システム、フォルダ構造の要点
- `CONTENT.md`: memory fragments と shadow prompts の内容

## Codexに任せやすい依頼

- 実装: `Phase 1を完了させて`
- 調査: `今の構成を見て、何が足りないか教えて`
- 修正: `ビルドエラーを直して`
- レビュー: `この差分をレビューして`
- 整理: `PHASE_CURRENT.mdを最新化して`
- テスト: `可能な範囲でビルドとテストを実行して`

## この環境でCodexができること

CodexはこのPCの作業フォルダを読み書きできます。

現在の作業フォルダ:

```text
C:\Users\che22\Documents\New project
```

主にできること:

- ファイルを読む
- ファイルを作成、編集する
- コマンドを実行する
- ビルドやテストを実行する
- ローカルアプリを起動して確認する
- 差分を確認する
- Git操作を補助する

注意:

- 作業フォルダ外への書き込みは制限されることがあります。
- 破壊的な操作は勝手にしません。
- ネットワークや外部ツールが必要な場合は許可を求めることがあります。

## Shadow Nurseryでの頼み方

このプロジェクトはフェーズ順に作る前提です。

よい依頼:

```text
Phase 1を始めて。AGENTS.mdのPhase 1以外は実装しないで
```

```text
Phase 1の受け入れ条件を満たしているか確認して
```

```text
Phase 2に進む前に、Phase 1のビルド確認と不足確認をして
```

避けたい依頼:

```text
全部まとめて作って
```

この仕様は「各フェーズを完成させてから次へ進む」ことが重要です。

## Codexに守らせる重要ルール

- フェーズ順に進める
- 作業前に仕様ファイルを読む
- 余計な機能を追加しない
- ビルド可能な状態を保つ
- ゲーム内テキストは短く、静かに、曖昧にする
- 影の正体を説明しない
- ジャンプスケア、ゴア、明るいRPG風UIを入れない
